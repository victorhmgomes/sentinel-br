Glossário · termos técnicos em linguagem simples

Orderbook
O "caderno de ofertas" de uma exchange: lista quem quer comprar (bids) e quem quer vender (asks) e a que preço. É a fotografia instantânea da liquidez.
Spread
Diferença entre o melhor preço de venda (ask) e o de compra (bid). Spread baixo = mercado saudável. Spread alto = mercado nervoso ou pouca liquidez.
Profundidade (depth)
Quanto volume existe em ordens próximas do preço atual (ex: ±2%). Profundidade caindo = liquidez sumindo → possível manipulação ou ataque em andamento.
Z-score
Quantos "desvios padrão" um valor está da sua média histórica. Z=6 significa 6× mais anômalo que o normal. Bom pra transformar "está estranho" em número objetivo.
Imbalance (desequilíbrio)
Número entre −1 e +1 que resume se o livro tem mais compradores (bid-heavy, positivo) ou vendedores (ask-heavy, negativo). +0,5 = muito mais compra do que venda.
Decouple
Quando um ativo que "deveria" seguir um preço de referência se solta dele. Ex: USDT-BRL deveria = BTC-BRL / BTC-USDT. Se fica 1% acima, decouplou.
FX implícito
Taxa de câmbio deduzida por fórmula, não negociada diretamente. Ex: divide o preço do BTC em reais pelo preço do BTC em dólares — dá uma estimativa do "dólar" implícito naquele momento.
VWAP (Volume-Weighted Average Price)
Preço médio que você pagaria se executasse uma ordem grande de uma vez. Quanto maior a ordem, mais o preço piora (slippage). Mostra o custo real de mover volume.
Smurfing
Dividir uma quantia grande em várias pequenas (ex: R$ 100k → 10× R$ 10k) pra evitar gatilhos de monitoramento. Clássico em lavagem pós-fraude.
Layering
Fase 2 da lavagem de dinheiro: converter o valor entre vários ativos/venues pra "apagar" a trilha. Ex: PIX → USDT → BTC → USDT em outra wallet.
Cash-out
Retirar valor do sistema criminal. No Brasil costuma ser: PIX roubado → conta-laranja → exchange BR → cripto → cold wallet no exterior.
VASP
Virtual Asset Service Provider. É o termo regulatório oficial pra exchange de cripto, custodiante, corretora cripto, etc.
Backtest
Rodar o sistema contra dados do passado pra ver se teria pegado eventos conhecidos e quantos "falsos alarmes" teria gerado. Base pra calibrar thresholds.
F1
Nota de 0 a 1 que combina precisão (dos meus alertas, quantos são reais?) e recall (dos ataques reais, quantos peguei?). F1=1 é perfeito, F1=0.5 é ruim.
Threshold
Limiar de corte. "Só dispara alerta se Z-score ≥ 6" — 6 é o threshold. Subir diminui falsos positivos mas pode deixar ataques passarem.
TP / FP / TN / FN
True positive (alarme real), false positive (alarme falso), true negative (silêncio correto), false negative (ataque não detectado).
Stablecoin
Cripto atrelada a uma moeda (USDT = dólar, BRL1 = real). É o "estacionamento" da lavagem porque preserva valor durante o layering.
Mula (money mule)
Pessoa usada pra movimentar dinheiro ilícito sem saber (ou sabendo). Conta laranja, essencialmente. Detectamos quando várias mulas fazem o mesmo movimento em venues diferentes no mesmo dia (ring-of-mules).
Pré-spike BR
Sinal onde uma exchange brasileira se move antes da Binance global. Indica pressão local (típico de cash-out PIX→cripto) antes da arbitragem fechar o gap.
Correlator / super-alerta
Camada que agrega alertas isolados em "eventos" quando ≥2 detectores diferentes disparam no mesmo ativo numa janela curta. Reduz muito o falso-positivo.

Metodologia. N snapshots reais são coletados do orderbook ao vivo (Binance/MB/Foxbit). FP é medido leave-one-out: cada snap vira o "agora" com os demais como histórico. TP é medido injetando 6 assinaturas sintéticas (spread, depth drop, price burst, divergência, pré-spike local, descolamento USDT-BRL) em intensidades crescentes. Para cada threshold do grid: precisão = TP/(TP+FP), recall = TP/total_de_ataques, F1 = média harmônica. A recomendação pega o menor threshold com FP ≤ alvo, maximizando recall sem inflar falso-positivo.

Como funciona. Cada detector dispara alertas isolados. O correlator agrupa alertas do mesmo ativo em janela deslizante (fecha o evento após cooldown sem alerta novo). Dentro do evento, repetições do mesmo detector são dedupadas. A severidade final escala com a convergência: 2 detectores → medium, 3 → high, 4+ → critical. A intuição: quando múltiplos sinais independentes concordam, a probabilidade de FP cai drasticamente.

Como ler: soma-se em BRL a liquidez dentro de ±0,5% do mid consolidado em cada venue BR. Verde = compradores prontos (bid); vermelho = vendedores prontos (ask). Um livro saudável fica perto de 50/50. Fuga para stablecoin (USDT/BTC)

Proxy de layering pós-incidente

O que é: razão diária vol(USDT-BRL) / vol(BTC-BRL) no Mercado Bitcoin (maior liquidez de USDT-BRL no país). Por que importa: fraudador prefere USDT porque preserva valor durante o layering. Um pico inédito dessa razão = rotação massiva para stablecoin (clássico pós-incidente). Foxbit e NovaDAX podem ser adicionados como validação cruzada — se os 3 venues subirem juntos, é sinal sistêmico; se só MB sobe, é idiossincrático do venue.

Premium BR vs. global

(BTC-BRL)/(BTCUSDT × USDT-BRL) − 1

O que é: preço BRL efetivo / preço global sintético. Por que importa: R$350M forçados no book BR deslocam o preço doméstico do fair value global. Arb bots fecham em segundos, então um prêmio persistente acima de 1% em base diária é pressão estrutural. Premium negativo sinaliza venda forçada (distressed flow, p.ex. congelamento de conta).

Concentração BR: MB × Foxbit

Participação no volume BTC spot diário

O que é: fatia diária de MB vs. Foxbit no volume BTC-BRL. Por que importa: quedas abruptas na fatia dominante sugerem dispersão do cash-out entre venues para fugir da monitoração de um único VASP. Típico em fases de smurfing entre múltiplas corretoras brasileiras.

Por que serve para fraude: uma frota de mulas tipicamente opera em 2+ exchanges para diluir o ticket por VASP e fugir de limites. Quando 3+ séries BR disparam Z≥2 no mesmo dia, a probabilidade de ser ruído individual cai drasticamente. Base rate aqui é a referência: dias-flag / dias totais. Em produção (intra-day + co-depósito on-chain) esse base rate cai 5–10× e vira alerta de alta confiança.

O que é: taxa periódica (a cada 8h) que longs pagam a shorts (ou vice-versa) para manter o perpétuo perto do spot. Sinal anti-fraude: funding fortemente negativo de forma persistente sinaliza pressão vendedora estrutural — clássico em cash-out forçado pós-incidente, ou hedge defensivo de uma exchange/desk grande que recebeu fluxo suspeito. Um funding negativo isolado é ruído; combine com pico de volume USDT-BRL no MB para alta convicção.

Leitura honesta: com granularidade diária, o base rate de alertas é alto (35% dos dias têm pelo menos 1 alerta). Isso faz com que "ter alerta em ±3d" seja pouco discriminativo — janela aleatória já teria 95% de hit. Sobre o protótipo

Todos os gráficos e alertas acima foram gerados por scripts/fetch_data.py + scripts/detect_anomalies.py usando APIs públicas (Binance api.binance.com/api/v3/depth, Mercado Bitcoin api/v4/orderbook, Foxbit rest/v3/markets/orderbook, NovaDAX api.novadax.com/v1/market/depth, BitPreço api.bitpreco.com/{pair}/orderbook, Ripio Trade api.ripiotrade.co/v4/public/orders/level-2). Não houve acesso a dados proprietários das exchanges. Para um produto operacional recomenda-se: (i) conectores autenticados (VASP API) para orderbook L2/L3 e fluxo de depósito/saque; (ii) inteligência on-chain para atribuição de carteiras; (iii) integração com PIX/BC+ para correlacionar entrada fiat com saída cripto; (iv) modelos supervisionados treinados sobre casos rotulados de cash-out e mulas.
